Comparison of Norman’s Action Cycle and GOMS Concepts
Norman’s Stage Real-World Example (Automatic Syringe Filling Machine) GOMS Interpretation & Rationale (GOMS 1983 vs Norman 1988) Alignment Analysis
Goal ↔ GOAL (User Intention) Nurse wants to set correct medicine dose (e.g., 1372 units). Goal: Set required dose value before injection.
Top-level objective in GOMS. 		Perfect Match — Both models start with user intention to achieve system outcome.
Plan ↔ METHOD & Selection Rules Nurse decides how to enter dose value.
Old Interface: Type full number using numeric keypad.
Improved Interface: Adjust each digit using increment/decrement buttons.
Nurse prefers safer method to reduce risk of error. 		Method 1: Enter digits via keypad.
Method 2: Adjust digits individually using + / – controls.
Selection Rule: If high-risk medication → choose digit adjustment method. 		Conceptual Match — Norman calls it planning; GOMS models alternative methods and rule-based selection.
Specify (Forming action sequence & locating controls) Nurse visually locates dose input interface and determines which digits must change. Not explicitly modeled in classic GOMS. Often abstracted as mental preparation (KLM M operator). Key Divergence — Norman emphasizes perception and cognition; GOMS abstracts this step.
Execute ↔ OPERATOR (Physical interaction) Nurse presses keypad digits or increment/decrement buttons.
Example actions: Press button, move finger between controls. 		Operators: Keystroke (K), Pointing (P), Mental preparation (M).
Example KLM sequence: [M] Decide digit → [P] Move → [K] Press button. 		Direct Match — Both models explicitly describe physical user actions.
Perceive / Interpret / Evaluate (Feedback Loop) Nurse checks system feedback:
1. Perceive updated digits on display.
2. Interpret whether displayed dose is correct.
3. Evaluate whether machine is ready for injection.
If wrong → repeat adjustment. 		Not modeled in classic GOMS. Assumes perfect perception and immediate success. Fundamental Divergence — Norman focuses on closing the Gulf of Evaluation; GOMS focuses on task efficiency.
